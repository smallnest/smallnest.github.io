Scalaзј–зЁӢзҡ„иӣӢзі•жЁЎејҸе’Ңдҫқиө–жіЁе…Ҙ

еҰӮжһңдҪ жҳҜдёҖдёӘJavaејҖеҸ‘иҖ…пјҢзҶҹжӮү дҫқиө–жіЁе…Ҙ жЁЎејҸ, ж·ұеәҰдҫқиө–SpringжЎҶжһ¶зҡ„иҜқпјҢеңЁдҪҝз”ЁScalaеҒҡејҖеҸ‘ж—¶пјҢдјҡй—®дёҖдёӘй—®йўҳпјҢеңЁScalaдё–з•ҢйҮҢпјҢеҰӮдҪ•е®һзҺ°зұ»дјјSpringжЎҶжһ¶зҡ„дҫқиө–жіЁе…Ҙе‘ўпјҹ

е°Ҫз®ЎеҮҪж•°ејҸзј–зЁӢзҡ„дҝЎеҫ’и®Өдёәд»–д»¬дёҚйңҖиҰҒDIжЎҶжһ¶пјҢй«ҳйҳ¶(high-order)еҮҪж•°и¶іеӨҹдәҶгҖӮдҪҶжҳҜеҜ№дәҺеҗҢж—¶ж”ҜжҢҒйқўеҗ‘еҜ№иұЎзҡ„зј–зЁӢе’ҢеҮҪж•°ејҸзј–зЁӢзҡ„ScalaжқҘиҜҙпјҢдҫқиө–жіЁе…ҘжҳҜеҫҲеҘҪзҡ„е®һзҺ°еә”з”Ёзҡ„дёҖз§Қи®ҫи®ЎжЁЎејҸгҖӮ
иӣӢзі•жЁЎејҸ(Cake pattern)жҳҜScalaе®һзҺ°дҫқиө–жіЁе…Ҙзҡ„ж–№ејҸд№ӢдёҖгҖӮ
иӣӢзі•жЁЎејҸжҳҜ Scala д№ӢзҲ¶ Martin Odersky еңЁе…¶и®әж–Ү Scalable Component Abstractions дёӯйҰ–е…ҲжҸҗеҲ°гҖӮ

д»Җд№ҲжҳҜиӣӢзі•жЁЎејҸе‘ў? дёҖдёӘйқһжӯЈејҸзҡ„дҪҶжҳҜеҫҲеҪўиұЎзҡ„и§ЈйҮҠжҳҜпјҡ

  • иӣӢзі•жңүеҫҲеӨҡйЈҺе‘іпјҢ дҪ еҸҜд»Ҙж №жҚ®дҪ зҡ„йңҖиҰҒеўһеҠ зӣёеә”зҡ„йЈҺе‘ігҖӮдҫқиө–жіЁе…Ҙе°ұжҳҜеўһеҠ и°ғж–ҷгҖӮ
  • иӣӢзі•жңүеҫҲеӨҡеұӮ (layer)гҖӮеҰӮжһңдҪ жғіиҰҒдёҖдёӘжӣҙеӨ§зҡ„иӣӢзі•дҪ е°ұеҸҜд»ҘеўһеҠ жӣҙеӨҡзҡ„еұӮгҖӮ

жҲ‘д»¬е…Ҳд»ҘSpringеёёз”Ёзҡ„Userж•°жҚ®еә“иҜ»еҸ–зҡ„е®һзҺ°дёәдҫӢпјҢзңӢзңӢScala йЈҺж јзҡ„дҫқиө–жіЁе…Ҙ(иӣӢзі•жЁЎејҸ)жҳҜеҰӮдҪ•е®һзҺ°зҡ„гҖӮ

Java/SpringйЈҺж јзҡ„дҫқиө–жіЁе…Ҙе®һзҺ°

дёҖдёӘдј з»ҹзҡ„Springе®һзҺ°жҳҜе°ҶзЁӢеәҸеҲ’еҲҶдёә"Repository"еұӮ(DAO layer) е’ҢServiceеұӮгҖӮ

жҲ‘д»¬еҸҜиғҪжңүеӨҡдёӘUserRepositoryзҡ„е®һзҺ°пјҢ жҜ”еҰӮJPA, JDBC, Hibernate, iBATIS зӯүпјҢз„¶еҗҺе°Ҷе…·дҪ“зҡ„е®һзҺ°йҖҡиҝҮSpringжіЁе…ҘеҲ° UserServiceдёӯгҖӮ
иҝҷйҮҢжҲ‘д»¬з”ЁдёҖдёӘMockжқҘз®ҖеҚ•иҝҷдәӣиҝҷдёӘTrait, з„¶еҗҺжЁЎжӢҹSpringжіЁе…ҘпјҢ(Springдјҡж №жҚ®й…ҚзҪ®е’ҢReflectиҮӘеҠЁе®һзҺ°е®һдҫӢеҢ–е’ҢжіЁе…ҘпјҢиҝҷйҮҢжҲ‘д»¬еҸӘжҳҜжЁЎжӢҹе…¶еҺҹзҗҶпјҢе№¶жІЎжңүдҪҝз”ЁSpringжЎҶжһ¶)пјҡ

Scala/Cake patternе®һзҺ°дҫқиө–жіЁе…Ҙ

е’ҢдёҠйқўзҡ„д»Јз Ғзұ»дјјпјҢжҲ‘д»¬жңүдёүдёӘclass/trait: UserRepository , MockUserRepository е’Ң UserService пјҢ е…¶дёӯ MockUserRepository жҳҜ UserRepository зҡ„е…·дҪ“е®һзҺ°пјҢ
зҺ°еңЁжҲ‘д»¬жғіжҠҠ MockUserRepository жіЁе…ҘеҲ° UserService гҖӮжіЁж„Ҹ UserService е’Ң UserRepository зӣ®еүҚжІЎжңүд»»дҪ•дҫқиө–е…ізі»гҖӮ

Scalaзҡ„иӣӢзі•жЁЎејҸдёӯжҲ‘д»¬йңҖиҰҒеЈ°жҳҺеҮ дёӘ Component пјҡ

иҝҷйҮҢдҪҝз”Ё self-type annotation еЈ°жҳҺ UserServiceComponent йңҖиҰҒ UserRepositoryComponent ( this: UserRepositoryComponent => )гҖӮ еҰӮжһңйңҖиҰҒеӨҡдёӘдҫқиө–пјҢеҸҜд»ҘдҪҝз”ЁдёӢйқўзҡ„ж јејҸ:

еү©дёӢзҡ„е°ұжҳҜжіЁе…ҘдәҶпјҢз”ҹжҲҗдёҖдёӘ ComponentRegistry еҜ№иұЎпјҡ

жҢәжјӮдә®зҡ„е®һзҺ°пјҢ еҰӮжһңзӣёеә”зҡ„дҫқиө–жІЎжңүжҸҗдҫӣпјҢжҲ–иҖ…жӢјеҶҷй”ҷиҜҜзӯүпјҢзј–иҜ‘ж—¶иғҪз«ӢеҲ»жҸҗзӨәжҲ‘д»¬гҖӮ
иҝҷиҝҳжңүдёҖдёӘеҘҪеӨ„е°ұжҳҜжүҖжңүзҡ„еҜ№иұЎйғҪжҳҜ val зұ»еһӢзҡ„гҖӮ

иҝҷж ·дҪ е°ұеҸҜд»ҘйҖҡиҝҮ ComponentRegistry.userService жқҘдҪҝз”ЁйЎ¶еұӮзҡ„з»„д»¶дәҶгҖӮ

иҝҷж ·жҲ‘д»¬еҸҜд»Ҙе®һзҺ°дёҖз§Қ"е№ІеҮҖ"зҡ„ж–№ејҸе®һзҺ°дёҚеҗҢзҡ„з»„д»¶жіЁе…ҘпјҢжҜ”еҰӮжҲ‘д»¬жғіеҚ•е…ғжөӢиҜ• userService пјҢе…¶дёӯе®ғзҡ„дҫқиө– userRepository йҖҡиҝҮmockзҡ„ж–№ејҸжҸҗдҫӣпјҡ

еҸҜд»ҘеҫҲж–№дҫҝзҡ„дёәжөӢиҜ•е®һзҺ°ж–°зҡ„дҫқиө–жіЁе…ҘпјҢе…¶дёӯ userRepository з”ұmockе®һзҺ°гҖӮ

иҝҷе°ұжҳҜиӣӢзі•жЁЎејҸзҡ„е®һзҺ°гҖӮйҖҡиҝҮ component trait жҠҪиұЎжҺҘеҸЈе’Ңдҫқиө–пјҢжңҖеҗҺеңЁдёҖдёӘ ComponentRegistry жіЁе…Ҙеҗ„дёӘе…·дҪ“зҡ„е®һзҺ°гҖӮ
