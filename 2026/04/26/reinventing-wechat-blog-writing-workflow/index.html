写公众号最痛苦的那段流程，终于有人重做了

写公众号最耗人的，常常不是内容本身，而是写完之后那一整套排版、预览、代码块、图片、复制发布的返工流程。wxeditor 不是普通 Markdown 编辑器，而是专门为微信公众号排版与发布场景重做的一套工具链。

大部分人苦微信公众号文章编辑器久矣！例如去年宝玉大佬的总结：

文章写完了，真正折磨人的事才刚刚开始。

标题不好看，要改!

代码块太丑，要改!

图片贴进去又乱了，要改!

好不容易排完，粘到公众号后台以后样式又变了，还得再改。

如果你经常写公众号，你一定懂这种崩溃感。

很多人以为公众号创作最难的是写。

其实不是。

真正耗时间、耗耐心、耗状态的，往往是写完之后那一堆排版和发布动作。

而最近看到的这个工具，我觉得非常值得写公众号的人认真看一眼。

它叫 wxeditor, 网址是 https://md.rpcx.io

不是“又一个 Markdown 编辑器”。

而是一个把公众号作者最烦的那段流程，重新做了一遍的工具。

它最厉害的，不是让你能写 Markdown

因为现在会写 Markdown 的工具太多了。

真正稀缺的，是一个能够把这条链路理顺的工具：

写作 -> 预览 -> 排版 -> 图片处理 -> 复制 -> 粘贴到公众号后台

wxeditor 做的，就是这件事。

它不是只给你一个编辑框。

它是把公众号作者最常遇到的问题，一刀一刀拆开，再一刀一刀补上。

第一件事：它把工作流变顺了

现在这个工具的核心结构是三栏：

  • 左边编辑区
  • 中间预览区
  • 右边样式面板

这不是简单的界面分栏。

它真正解决的是：你不用在多个工具之间来回跳。

更关键的是，它还把预览这件事做得更贴近真实发布场景：

  • 编辑区和预览区支持滚动同步
  • 默认是手机宽度预览
  • 折叠编辑区后还能切换到 1024px 网页预览模式

这意味着，你在排版长文时，不再需要一边拉滚动条，一边猜最终效果。

很多返工，其实从这里就已经开始减少了。

第二件事：它把那些高频但烦人的小动作一起做了

真正高频写作的人都知道，最拖效率的从来不是某一个大问题。

而是那些每次都会碰到的小动作：

  • Markdown 格式不统一
  • 外链处理麻烦
  • 想换个风格又得从头调
  • 排着排着发现整体气质不对

wxeditor 现在已经把这些动作做成了直接可用的能力：

  • Markdown 格式化
  • 外链转脚注
  • 随机色系
  • 随机风格
  • 一键复制到公众号
  • 复制 HTML 源码

这些单看都不算“炫技”。

但它们一起出现，说明这工具不是只做了个壳。

它是在认真压缩你发布前的摩擦。

第三件事：它不是给你几个模板，而是给你一整套主题系统

这点我觉得特别重要。

很多排版工具的主题，本质上只是几套固定皮肤。

第一篇觉得新鲜，第二篇就腻了。

免费的一些编辑器只提供几套上个时代的风格，离顶尖的风格差远了。

但 wxeditor 这次在主题系统上明显做得更深：

  • 内置多组公众号风格主题
  • 还有一组公司风格主题，采用DESIGN.md的分析，精准采集世上顶尖公司的网站风格
  • 160套代码风格任意可以，实时预览
  • 支持自定义主题保存
  • 支持重命名和删除自定义主题
  • 支持 JSON 导入 / 导出

这意味着它不是只想让你“换一个样子”。

它是想让你真正沉淀自己的排版风格。

对于高频创作者来说，这件事会直接影响长期效率。

因为一旦你把风格稳定下来，后面每一篇文章都会更快。

第四件事：标题系统终于像样了

公众号文章的“精致感”，往往不是从正文开始体现的。

而是从标题层级开始。

标题一乱，整篇文章立刻显得散。

标题一稳，整篇文章的气质立刻不一样。

wxeditor 在标题这件事上做了非常多补强：

  • 新增多批 SVG 标题装饰
  • H1 / H2 / H3 / H4 的装饰变化更丰富
  • H3 的“点尾线”还重新做过，收尾更完整
  • H2 / H3 / H4 支持 Unicode 前缀装饰

你现在可以直接自己输入字符，给标题加前缀，比如：

  • ✦
  • ➜
  • ◆
  • 【
  • ▍

而且不是写死在模板里的。

是你自己输入，自己看效果，自己决定哪一种最适合这篇文章。

这对那些在意版式风格的人来说，真的很有价值。

第五件事：代码块终于不再像附属品了

如果你写技术文章、AI 内容、开发教程，这一块你一定会在意。

因为太多工具的代码块只是“有这个功能”，但实际并不好用。

wxeditor 这次把代码块和代码高亮彻底理顺了：

  • pre 预设负责容器背景、边框、圆角和整体气质
  • highlight.js 160套主题负责 token 语法高亮

这听起来像实现细节。

但用户侧的感受很直接：

代码块终于不是假切换了。

你换容器风格，它真的变。

你换代码主题，它真的变。

而且这次还新增了多套 dark 代码块预设：

  • 午夜蓝
  • 石墨灰
  • 夜林
  • 黑曜
  • 余烬暗棕
  • 深海

如果你平时写代码、贴配置、发工作流，这部分会非常常用。

第六件事：代码主题切换，也终于被做对了

这一点我非常想单独夸一下。

现在顶部工具栏里，多了一个独立的“代码主题”下拉框。

它接的不是单一固定主题，而是 highlight.js 官方 examples 里的160套漂亮的主题体系。

但更重要的不是“主题多”。

而是它做了几件非常对的体验优化：

  • 代码主题支持切换
  • 代码主题支持持久化保存
  • 代码主题名称过长会自动截断，避免顶栏太乱
  • 浅色代码块只显示 light 主题
  • 深色代码块只显示 dark 主题

这意味着它不是把所有开关都扔给你。

它开始替你做合理筛选，主动避免错误搭配。

这种设计，一看就不是“功能越多越好”的思路。

而是非常贴近真实使用场景的思路。

第七件事：图片链路，它是完整接住了

很多工具会说自己支持图片。

但真正麻烦的，从来不是“能不能插图”。

而是后面这一整串事：

  • 能不能直接粘贴
  • 能不能拖拽上传
  • 会不会太大
  • 会不会丢
  • 刷新后还在不在
  • 复制到公众号时还能不能恢复

wxeditor 这部分做得很完整：

  • 支持粘贴图片
  • 支持拖拽上传
  • 自动压缩
  • 用 img:// 做占位
  • 用 IndexedDB 本地持久化图片
  • 预览时恢复图片
  • 复制到公众号时也恢复图片

这一点对高频写作的人来说，不是小优化。

而是决定体验是不是稳定的关键。

第八件事：它已经不是“演示级工具”，而是能真正进入工作流的工具

判断一个工具是不是能长期用，最简单的方法就是看它有没有认真处理“状态保存”。

wxeditor 这次把这一层也补齐了：

  • 编辑内容自动保存
  • 样式设置自动保存
  • 代码主题自动保存
  • 自定义主题自动保存
  • 配置还能导入 / 导出 JSON

这说明它不只是想让你“试试看”。

它已经在往“真正进入你日常工作流”这个方向走。

所以你现在应该能明白，为什么我会觉得它值得被大量公众号作者收藏

因为它不是单点做得花。

而是把公众号作者最烦、最碎、最耗状态的那一整段流程，系统性地补了一遍：

  • 三栏工作流
  • 实时预览
  • 滚动同步
  • 网页预览模式
  • Markdown 格式化
  • 外链转脚注
  • 大量主题
  • 自定义主题
  • SVG 标题装饰
  • Unicode 标题前缀
  • dark 代码块预设
  • 代码高亮主题切换
  • light / dark 智能过滤
  • 图片压缩
  • 图片本地持久化
  • JSON 导入 / 导出
  • 一键复制到公众号

你把这些放在一起看，它解决的根本不是“排版漂不漂亮”。

它解决的是：

一篇公众号文章，从写完到发出，中间到底要不要被反复折腾。

最后一句

如果你只是偶尔写一篇，
那 wxeditor 可能只是一个不错的工具。

但如果你是：

  • 高频公众号作者
  • 技术内容创作者
  • AI 内容创作者
  • 习惯用 Markdown 写作的人
  • 已经受够排版返工的人

那它大概率不是“不错”。

而是：

你会越来越离不开的那种工具。
